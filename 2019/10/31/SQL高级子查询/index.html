SQL高级子查询
Posted on 2019-10-31   Updated on 2019-10-31

1.简单子查询

(1)什么是子查询？

所谓子查询，是指包含在一个INSERT、DELETE、UPDATE、SELECT命令中的SELECT查询。

也可以理解为增删改查命令中嵌套着查询命令。

(2)子查询在WHERE语句中的一般用法：

SELECT … FROM 表1 WHERE 字段1 >(子查询)，外面的查询称为父查询，括号中嵌入的查询称为子查询 ，UPDATE、INSERT、DELETE一起使用，语法类似于SELECT语句

将子查询和比较运算符联合使用，必须保证子查询返回的值不能多于一个；

2. 高级子查询

(1)EXISTS子查询

如果子查询的结果非空，即记录条数1条以上，则EXISTS （子查询）将返回真（true），否则返回假(false) ，EXISTS也可以作为WHERE 语句的子查询，但一般都能用IN子查询替换

通过在子查询中使用EXISTS子句，可以对子查询中的行是否存在进行检查

(2)使用IN和NOT IN的子查询

1.IN子查询

IN子查询后面可跟随返回多条记录的子查询，用于检测某列的值是否在某个范围；

–使用IN后面的子查询可以返回多条记录。常用IN替换等于(=)的比较子查询

IN：在指定项内，同 IN(项1,项2,…)。

2.NOT IN 子查询

–查询没有参加考试的学员、姓名？

注意：

‘IN’ 是 ‘= ANY’ 的简写，’NOT IN’ 是 ‘<> ALL’ 的简写;

(3)使用ALL,ANY/SOME的子查询

在使用子查询语句的过程中经常使用ANY、SOME、ALL（另外还有一个IN）关键字作为筛选条件。

ANY，SOME，ALL关键字的使用

1.ANY：与比较操作符联合使用，表示与子查询返回的任何值比较为 TRUE ，则返回 TRUE 。

2.ALL：与比较操作符联合使用，表示与子查询返回的所有值比较都为 TRUE ，则返回 TRUE 。

特殊情况:

如果 table2 为空表，则 ALL 后的结果为 TRUE；

3.SOME：ANY 的别名，较少使用。
